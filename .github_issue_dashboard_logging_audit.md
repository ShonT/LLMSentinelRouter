# Dashboard LoggingAudit Error - Root Cause Analysis and Fix

## Issue
Dashboard endpoint `/api/dashboard/metrics` was generating recurring error:
```
Failed to load request logs: LoggingAudit.__init__() missing 1 required positional argument: 'db_session'
```

## Root Cause

**File**: `sentinelrouter/sentinelrouter/dashboard.py` (lines 1512-1522)

**Problem**: The `get_routing_logs()` function was attempting to instantiate `LoggingAudit` without passing the required `db_session` parameter:

```python
if include_preview:
    try:
        audit = LoggingAudit()  # ❌ Missing db_session parameter
        for log in logs[:20]:
            if log.request_id:
                log_data = audit.read_request_log(log.request_id)  # ❌ Method doesn't exist
```

**Additional Issue**: The code was also calling `audit.read_request_log()` which is a method that doesn't exist in the `LoggingAudit` class. The class only has write methods, not read methods.

## Fix Applied

Replaced the broken preview loading logic with a TODO placeholder since the feature is incomplete:

```python
if include_preview:
    # TODO: Implement read_request_log method in LoggingAudit class
    # Currently LoggingAudit only writes logs, doesn't have a read method
    # Skipping preview feature until read functionality is implemented
    pass
```

## Impact

- **Before**: Dashboard `/api/dashboard/metrics` endpoint logged errors every ~5 seconds (on every dashboard poll)
- **After**: No errors, dashboard functions normally
- **Feature Loss**: Preview functionality was never working anyway (method didn't exist), so no actual functionality was lost

## Future Work

To properly implement the preview feature:

1. Add `read_request_log(request_id: str) -> Optional[Dict]` method to `LoggingAudit` class
2. Method should read from the JSON files in `logs/requests/` directory
3. Method should parse the filename pattern: `{timestamp}_{model}_{cost}.json`
4. Search for files matching the request_id
5. Return parsed JSON data or None if not found
